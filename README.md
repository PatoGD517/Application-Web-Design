**Markdown Description**
Markdown is a lightweight markup language used for formatting text. It allows you to create formatted text using a plain text editor. Markdown is commonly used for README files, documentation, and writing content for web pages.

# Markdown Tagging Options

Markdown offers a variety of options for tagging and formatting text. Here are some commonly used ones:

- **Headings**: Use `#` for H1, `##` for H2, `###` for H3 and so on.
- **Bold Text**: Wrap text in double asterisks `**bold**` or double underscores `__bold__`.
- **Italic Text**: Wrap text in single asterisks `*italic*` or single underscores `_italic_`.
- **Blockquotes**: Start lines with a `>` symbol.
- **Ordered Lists**: Use numbers followed by a period `1. Item`.
- **Unordered Lists**: Use asterisks `*`, plus signs `+`, or hyphens `-` to create bullet points.
- **Code Blocks**: Wrap inline code in backticks `` `code` `` or triple backticks for code blocks.
- **Links**: Use `[Link Text](URL)`.
- **Images**: Use `![Alt Text](Image URL)`.
- **Horizontal Rules**: Use three or more dashes `---` or asterisks `***`.
- **Tables**: Use pipes `|` to create tables with headers separated by hyphens `-`.

# Git Commands

## Check the status of a local repository.
git status

## Add individual files to the staging area
git add <filename>

## Add all changes globally to the staging area
git add .

## Commit changes with a message
git commit -m "Your commit message"

## Upload your changes to the remote repository
git push

## Create a new branch
git branch <branch-name>

## Switch to another branch
git checkout <branch-name>

## Create and switch to a new branch
git checkout -b <branch-name>

## Delete a branch
git branch -d <branch-name>

## Delete a branch forcefully
git branch -D <branch-name>

## List all branches
git branch

## Merge a branch into the current branch
git merge <branch-name>

## Roll back a repository to a specific commit
git reset --hard <commit-hash>

## Show commit history
git log

## Pull changes from a remote repository
git pull

## Clone a remote repository
git clone <repository-url>
